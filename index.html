Audio Privacy Guarantees
🔒
getUserMedia isolation Mic data stays in browser memory. No Web Speech API (which routes to Google servers).
🧵
AudioWorklet thread PCM frames processed on the audio render thread — never serialised or written to any socket or fetch call.
📦
ONNX WASM inference Transformers.js runs Whisper as WebAssembly. The model executes entirely in a WebWorker in this tab.
🚫
Zero audio bytes on wire Open DevTools → Network → filter "Media" or "XHR" — you will see zero audio requests during dictation.
✂️
Silero VAD on-device Speech segmentation uses a ~400 KB ONNX VAD model — no cloud endpoint.
Pipeline Architecture
1
getUserMedia → AudioContext
Mic captured at 16 kHz mono PCM. AudioContext taps raw Float32 frames.
getUserMedia({ audio:{sampleRate:16000, channelCount:1} })
2
AudioWorklet (audio render thread)
ScriptProcessorNode is deprecated. AudioWorklet runs on a dedicated thread — zero UI blocking, zero dropped frames.
process(inputs){ port.postMessage({pcm:inputs[0][0]}) }
3
Silero VAD via @ricky0123/vad-web
~400 KB ONNX model classifies each 30 ms frame. Only speech segments forwarded — prevents silence hallucinations.
onSpeechEnd: (audio) => transcribeLocally(audio)
4
Transformers.js Whisper (WebWorker)
ONNX-quantised Whisper runs in a Worker. WebGPU accelerated on Chrome/Edge. Model cached after first download.
pipeline('asr', 'onnx-community/whisper-tiny.en', {device:'webgpu'})
5
Text injection
Transcript appended to textarea. React/Vue inputs need a native input event to detect the value change.
dispatchEvent(new Event( 'input',{bubbles:true}))
Audio never leaves this browser tab — validate in DevTools → Network
